1 Showing what data went into a code block on export

Sometimes I define variables in the header of a code block and then use the code to analyze the data. In org-mode this is super, and you can read the file and easily see what is going on.

When you export the file, however, the information is lost, and in the exported result you cannot see what data went into a code block, or figure out where it is from.

Today we design an export filter to remedy this.


TBLNAME: tbl-data
x y
1 1
2 4
3 9

LANGUAGE: python
NAME: print-table

PARAMETERS: 
#+BEGIN_SRC python

#+name: print-table #+BEGIN_SRC python return data #+END_SRC 
return data

RESULTS: (print-table)
1 1
2 4
3 9

We are going to define a derived LaTeX backend to do this for us. This idea is based on the code shown in http://orgmode.org/manual/Advanced-configuration.html.


LANGUAGE: emacs-lisp
NAME: custom-export

PARAMETERS:

(defun my-table-format (table contents info)
  (let ((tblname (org-element-property :name table))
        (results (org-element-property :results table)))    
    (cond
     ((eq (elt (plist-get info :back-end) 2) 'html)  
      (concat
       (when tblname
         (format "<br>TBLNAME: <a name=\"%s\"></a>%s<br>" tblname tblname))
       (when results
         (format "<br>RESULTS: <a name=\"%s\"></a>%s<br>" results results))
       (org-html-table table contents info))))))


(defun my-src-block-format (src-block contents info)
  (let ((srcname (org-element-property :name src-block))
        (parameters (org-element-property :parameters src-block)))    

;; there seems to be a bug here. the parameters are not gathered.
;; furthermore, there do not seem to be any parameters!
    (save-excursion
      (goto-char (org-element-property :begin src-block))
      (re-search-forward "#\\+BEGIN_SRC")
      (let ((start)(end))
        (beginning-of-line)
        (setq start (point))
        (end-of-line)
        (setq end (point))
        (setq parameters (buffer-substring start end))))
    
    (cond
     ((eq (elt (plist-get info :back-end) 2) 'html)  
      (concat
       (format "<br>LANGUAGE: %s" (org-element-property :language src-block))
       (when srcname
         (format "<br>NAME: <a name=\"%s\"></a>%s<br>" srcname srcname))
       (when parameters
         (format "<br>PARAMETERS: <pre>%s</pre><br>" parameters))
(buffer-substring
       (org-element-property :begin src-block) (org-element-property :end src-block))
       (org-html-src-block src-block contents info))))))


(org-export-define-derived-backend 'my-html 'html
  :translate-alist '((table . my-table-format)
                     (src-block . my-src-block-format)))

(browse-url (org-export-to-file 'my-html "custom-src-table-export-2.html"))
